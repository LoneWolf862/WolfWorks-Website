Project

FPGA Tic-Tac-Toe

Digital Logic Design, FPGA Development, VGA Video Generation, and USB HID Interfaces

Completed / Functional Final Project

Short Description

A fully hardware-implemented Tic-Tac-Toe game developed on an FPGA platform using VGA video output and USB keyboard input. The project combined independently developed HDL modules for video generation, keyboard interfacing, game-state management, and verification into one integrated digital system.

Project Overview

FPGA Tic-Tac-Toe was developed as the final project for a digital systems and FPGA design course. Rather than selecting a minimal final project, our group wanted to build something that exercised multiple aspects of FPGA development and required meaningful subsystem integration.

The system used the FPGA's onboard VGA and USB interfaces to create a complete interactive Tic-Tac-Toe game. The game was displayed on an external monitor through a custom VGA video generator, while a USB keyboard provided player input.

Development was divided across the team. One teammate focused primarily on VGA video generation, another worked on game-state logic and simulation testbenches, and I developed the USB keyboard interface responsible for interpreting keyboard input and translating it into game commands.

Because FPGA development is fundamentally different from traditional software development, careful planning and interface standardization were critical. Each subsystem had to be designed independently while still integrating cleanly with the rest of the hardware design.

After roughly a week and a half of development, the team successfully integrated the major modules into a functional hardware-based game.

Features

  • FPGA-based hardware implementation of Tic-Tac-Toe.
  • Custom VGA video generation for monitor output.
  • USB keyboard input through the FPGA's onboard USB interface.
  • Numeric keypad position selection using keys 1–9.
  • Symbol selection controls using keyboard input.
  • Enter key confirmation for move placement.
  • Turn enforcement between players.
  • Hardware-implemented game-state management.
  • Modular HDL architecture with separately developed subsystems.
  • Simulation and verification through testbench development.
  • Collaborative integration between video, input, and game-logic modules.

Technical Challenges

  • Subsystem integration: independently developed HDL modules had to communicate through clean, standardized interfaces.
  • VGA timing: the display system required correct video timing and pixel generation to render the game board and symbols on an external monitor.
  • USB keyboard input: the input system needed to interpret keyboard data from the FPGA's onboard USB interface and convert it into usable game commands.
  • Keyboard decoding: numeric keypad input, symbol selection, and confirmation keys had to be recognized and mapped correctly.
  • Game-state control: the system had to track board state, enforce turn order, accept valid moves, and prevent invalid placement behavior.
  • Testbench verification: portions of the design were tested through simulation before being integrated onto hardware.
  • Team coordination: each module had to be developed separately while still matching the expected signals, timing, and behavior of the other modules.
  • Integration debugging: a small logic error occasionally caused an invalid character to appear in the Tic-Tac-Toe lattice instead of an expected X or O, requiring collaborative debugging across module boundaries.

Miscellaneous Notes

The value of this project was not simply the game itself. Tic-Tac-Toe was the visible demonstration, but the engineering challenge was building a complete digital system that connected USB keyboard input, input decoding, game-state logic, VGA video output, and hardware verification.

One of the most important lessons from this project was learning how to coordinate FPGA development across multiple contributors. Unlike many software projects, HDL modules require stricter interface definitions because hardware timing, signals, and state behavior must align correctly.

This project also provided practical experience working with external peripherals and real-world digital interfaces rather than purely simulated logic. Integrating USB input, VGA output, game-state management, and verification into a single design made the project significantly more complex than the simplicity of the game might suggest.
