Project

G.C.U

Goniometer Control Unit

Functional Prototype — Firmware incomplete at handoff

Short Description

The G.C.U. is an ESP32-based scientific instrumentation controller built to revive and operate an industrial goniometer salvaged from an old X-ray diffraction machine. It recreates the required motor control, sensor reading, safety, polarizer control, and PC communication interfaces needed to make the standalone mechanism usable again.

Project Overview

The Goniometer Control Unit came out of a university silicon research project. The university was disposing of an old X-ray diffraction system, and our group was able to recover the goniometer assembly. However, the goniometer had been removed from a much larger mechatronic machine and could not function by itself.

The original XRD system depended on a complex control stack, dedicated electronics, machine wiring, and advanced PC software. Once removed from that environment, the goniometer was essentially a heavy, precise mechanical assembly with no usable control interface.

After examining the unit, I identified the core controllable elements: two motorized axes, encoders, limit switches, and supporting sensor wiring. The goal became to build a standalone control interface that could recreate the necessary input/output behavior and allow the mechanism to be operated safely from a computer.

In addition to the original goniometer axes, the system was expanded to support a stepper-driven polarizer assembly. This allowed the orientation of the emitter polarizer to be adjusted under software control, giving the research setup an additional controllable experimental variable.

I designed the G.C.U. around an ESP32 microcontroller, motor drivers, a stepper motor driver, an internal power supply, a small display, USB-C serial communication, and an emergency stop. The controller was built into a dedicated enclosure and connected to the goniometer through the original dual DB-25 interface so the goniometer itself did not need to be permanently modified.

Features

  • ESP32-based motion and sensor control platform.
  • Designed to operate an industrial goniometer salvaged from an XRD system.
  • Controls two motorized goniometer axes through dedicated motor drivers.
  • Integrated support for a stepper-driven emitter polarizer.
  • Software-controlled polarizer orientation adjustment.
  • Reads encoder and limit switch feedback for basic position and safety awareness.
  • USB-C serial connection to a PC-based Python GUI.
  • Small onboard display for local status feedback.
  • Emergency stop button for immediate shutdown of motion hardware.
  • Dedicated enclosure containing power supply, motor drivers, controller, display, and wiring.
  • Dual DB-25 cable interface matching the original goniometer connections.
  • Preserves the physical integrity of the recovered goniometer by avoiding permanent modification.

Technical Challenges

  • Reverse engineering: the goniometer was recovered without the original control electronics, requiring its wiring, motors, encoders, and limit switches to be identified and mapped.
  • Interfacing with legacy hardware: the unit connected through two DB-25 connectors, so the new control unit had to preserve and reuse those existing interfaces.
  • Industrial motion safety: the goniometer is a heavy steel mechanism with powerful motors and gearing, so emergency stop capability was a necessary part of the design.
  • Standalone operation: the controller had to replace functions originally handled by a much larger XRD machine and PC software environment.
  • Sensor integration: encoders and limit switches had to be read reliably so the controller could track position after homing and prevent unsafe movement.
  • Polarizer integration: adding a third controlled axis for emitter polarization while maintaining compatibility with the original goniometer hardware.
  • PC control: a Python GUI was developed to send commands over USB serial and provide a more usable operator interface.
  • Non-destructive integration: the system was intentionally designed to connect through the original goniometer connectors rather than modifying the recovered mechanism.

Miscellaneous Notes

The G.C.U. was intended to make an otherwise useless recovered precision mechanism valuable again. Without the original XRD control system, the goniometer could not be operated, but its motors, encoders, gearing, and mechanical structure were still useful for research instrumentation.

The system was designed so the goniometer could be homed, then moved through controlled positions while reporting basic sensor and position data back to the controller. The long-term goal was to support precision wafer positioning and polarization control for silicon research work.

While the original goniometer provided the primary positioning axes, the project was expanded to include a stepper-driven polarizer assembly. This allowed the orientation of the emitter polarizer to be adjusted programmatically, providing an additional experimental variable that was not originally available through the recovered hardware alone.

I left the silicon research group before the firmware was fully completed. At handoff, the system was functional enough to move the motors, rotate the polarizer, and read basic sensor data, but not all final control features had been completed.
