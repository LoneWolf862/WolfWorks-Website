Project

VEDA

Legacy Infrastructure Platform

Operational / Ongoing Infrastructure Platform

Short Description

VEDA is a long-running infrastructure platform that evolved from a single gifted Dell PowerEdge server into a full family network, NAS, media, VPN, surveillance, and access-control ecosystem. It represents the first major infrastructure environment where I learned enterprise servers, storage systems, networking, self-hosted services, and long-term system maintenance.

Project Overview

VEDA began in 2016 with a Dell PowerEdge 1950 gifted to me by a contact in Navy IT. At first, it was simply a way to learn server hardware and build a basic NAS and Plex media server. Over time, that one server became the foundation for a much larger infrastructure platform.

The early server, known as Masurao, taught me how to work with enterprise hardware, configure storage, repair aging server components, install operating systems, set up Samba shares, run Plex, and understand the practical realities of keeping a server alive.

VEDA later expanded to include a larger Dell PowerEdge 2950 known as Gusion, TrueNAS storage, enterprise UniFi networking, structured Cat6A cabling, UPS-backed power, security cameras, access control, VPN connectivity, game servers, DNS management, and multiple hardware revisions.

Unlike a single finished project, VEDA is an infrastructure platform that grew through years of use, failure, troubleshooting, replacement, and redesign. It served real users while also teaching me the fundamentals of server administration, NAS design, networking, storage, security, and remote access.

The Story of VEDA

Masurao — First Server Platform

VEDA started with Masurao, a Dell PowerEdge 1950 that became my first real server. It was originally configured as a small NAS and Plex server for storing and streaming movies and TV. This system introduced me to enterprise hardware, server repair, operating system installation, Samba configuration, media encoding, RAID concepts, BIOS storage settings, and the basic discipline of running always-on infrastructure.

Masurao required a lot of repair and troubleshooting. I replaced CPUs, sourced compatible RAM, found a replacement power supply, repaired a fan module, and eventually configured an internal SATA SSD as a boot drive. That boot SSD dramatically improved performance and made the server much more usable.

Gusion — Dedicated NAS Expansion

After learning on the PowerEdge 1950, I later acquired a Dell PowerEdge 2950. This server became Gusion, a more capable NAS platform with better CPU options, more RAM capacity, and six hard drive bays instead of two.

Gusion was built around TrueNAS Core and became my primary storage system. During setup, I ran into a major compatibility limitation: the backplane could not properly support the 4 TB drives I had purchased and could only address them as 2 TB drives. Solving that issue required learning more about storage controllers, disk formatting, hardware limits, and practical NAS configuration.

UniFi Networking and the Birth of VEDA

VEDA became more than a server rack when my parents moved into a larger stone house with difficult wireless coverage. Consumer mesh networking struggled in the environment, so I proposed and implemented an enterprise-style UniFi network deployment.

I repurposed the existing telephone backbone into Ethernet infrastructure and ran dedicated shielded Cat6A lines to access point locations. This was my first serious experience with structured cabling, shielded patch panels, cable termination, grounding considerations, and enterprise-style network planning.

Around this time, the rack officially became known as VEDA. The network expanded to include a Dream Machine, access points, PoE switching, structured patching, and eventually camera and access-control systems.

UPS and Power Protection

Because the site had dirty power, outages, and occasional surges, I added a large UPS system to protect the network and servers. The UPS was integrated so that servers could be shut down safely after a sustained power outage, rather than simply losing power abruptly.

Surveillance and Access Control

As the UniFi ecosystem grew, VEDA expanded into surveillance and access control. I deployed a 24-port UniFi PoE switch, ran Cat6A infrastructure to camera locations, installed six cameras, and configured user accounts so family members could view and manage the system.

Later, the system was expanded again with UniFi access control and security hardware. This included door panels, security logic, and emergency contact configuration for fire and burglary events.

Game Servers and Compute Experiments

VEDA also became a place to experiment with hosting. An old PC rebuilt in a milk crate became an early game server for Minecraft and Terraria. Later, the original Masurao hardware was redesignated as Quantex-1, a remote compute experiment that was limited by age but useful as a learning platform.

Major Server Overhaul

After several years of use, I overhauled the platform. Masurao was rebuilt into an ASRock Rack-based system using a Ryzen 7 2700X, NVMe boot storage, multiple IronWolf drives, and a dedicated Quadro P6000 GPU for media encoding. This version moved deeper into TrueNAS, jails, containers, Plex hosting, and GPU acceleration.

The game server was also rebuilt into a dedicated 2U system known as Flauros, using a Ryzen 7 5700X and a Quadro P600 for basic acceleration. This system became a dedicated host for Minecraft and other community game servers.

Gusion 4.6 — Current Family Storage

Eventually, Gusion was upgraded again using a UniFi NAS Pro with eight new 4 TB NAS drives and mirrored NVMe cache drives. This became the family’s current cloud and file server, with user access, WireGuard connectivity, and integration into the broader UniFi-managed infrastructure.

Technical Challenges

  • Legacy hardware repair: keeping old PowerEdge servers functional through CPU, RAM, power supply, fan, and storage repairs.
  • Storage limitations: working around controller and backplane limits that prevented full use of larger NAS drives.
  • TrueNAS learning curve: learning storage pools, access control lists, shares, permissions, jails, and service management.
  • Media encoding: configuring Plex and later GPU-assisted encoding for a more capable media server.
  • Wireless coverage: designing Wi-Fi for a large stone house with difficult RF conditions.
  • Structured cabling: repurposing existing wiring and installing shielded Cat6A infrastructure.
  • UPS integration: protecting servers and coordinating safe shutdown during power outages.
  • Security systems: adding cameras, user permissions, access control, and emergency notification logic.
  • VPN access: providing secure remote access to family storage and internal resources.
  • Long-term maintenance: keeping an evolving infrastructure platform operational across multiple generations of hardware.

Miscellaneous Notes

VEDA is best understood as the infrastructure environment where I learned by doing. It taught me how to repair servers, build NAS systems, configure networks, manage users, deploy VPNs, work with cameras, integrate access control, and support real people using real services.

The platform has gone through many revisions, names, machines, and roles. Some parts were elegant, some were improvised, and some were lessons learned the hard way. That history is exactly why VEDA is valuable: it represents years of accumulated infrastructure experience.

VEDA continues to operate, though it now serves as the older, stable infrastructure platform while newer development and hosting work has moved to ARCHON.
