Project

C# Circuit Analyzer

Object-Oriented Programming, Circuit Modeling, and Console-Based Simulation

Completed / Functional Coursework Project

View GitHub Repository

Short Description

A console-based circuit simulator written in C# that allows users to construct circuits using named electrical nets, reduce simple equivalent component values, and analyze basic resistive loops. The project was originally created as a C# final project, but expanded beyond the basic class scope as a personal challenge.

Project Overview

The C# Circuit Analyzer was built around the idea of representing circuits through named nets instead of fixed visual layouts. Each component is defined by its value and the two nets it connects between, allowing the program to model different circuit topologies in a structured way.

The project supports voltage sources, resistors, capacitors, and inductors. It can identify and reduce simple series and parallel relationships, then perform basic analysis on valid simple resistive loops.

This project was also an exercise in object-oriented design. Components share a common base class, while individual component types implement their own behavior for series and parallel combination. This allowed the same reduction logic to operate across different electrical component types while still respecting their unique equations.

Features

  • Console-based C# circuit builder and analyzer.
  • Net-based circuit construction using named nodes.
  • Supports DC voltage sources, resistors, capacitors, and inductors.
  • Series and parallel equivalent value reduction.
  • Basic simple-loop resistive analysis using Ohm’s Law.
  • Object-oriented component model using abstraction, encapsulation, inheritance, and polymorphism.
  • Graceful error handling for unsupported or invalid circuit configurations.
  • Menu-driven console interface for adding components, viewing circuits, reducing values, and analyzing loops.

Technical Challenges

  • Net-based modeling: representing circuit connectivity through named nets rather than hardcoded layouts or fixed component positions.
  • Equivalent reduction: detecting simple series and parallel relationships across different component types.
  • Component-specific behavior: implementing different combination rules for resistors, capacitors, and inductors.
  • Analysis limits: intentionally restricting full circuit analysis to simple valid resistive loops to keep the project manageable.
  • Error handling: preventing unsupported topologies from crashing the program and instead returning useful diagnostic messages.
  • OOP structure: using inheritance and polymorphism so each component type could respond correctly to shared method calls.

Miscellaneous Notes

The project intentionally avoided full SPICE-like circuit solving. It does not perform AC analysis, transient analysis, full Kirchhoff solving, or matrix-based branch analysis. The goal was to build a clean, understandable, extensible foundation rather than a full professional simulator.

The most interesting part of the project was the net-based structure. By treating wires as named connection points, the program could support flexible user-defined circuits without requiring a graphical interface.

Future improvements could include saving and loading circuits, graph-based traversal, branch solving, a GUI interface, AC analysis, and time-domain simulation.
