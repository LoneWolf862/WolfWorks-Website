Project

Arcade Control Deck Interface

Custom PCB Design, USB HID, Arcade Controls, and Embedded I/O

Completed / Functional Prototype — Two hardware revisions

View GitHub Repository

Short Description

A custom Teensy 4.1-based arcade control interface PCB designed for BushWorks Entertainment to connect buttons, joysticks, and optical encoders to a Raspberry Pi over USB. The board replaced an unreliable ESP32 prototype and provided a faster, cleaner, more maintainable control system for arcade hardware.

Project Overview

This project was built for my friend’s business, BushWorks Entertainment. The arcade machine ran its game software on a Raspberry Pi, but needed a reliable interface between the physical control deck and the Pi. The existing prototype used an ESP32, which worked acceptably for normal button presses but struggled with the two optical encoders used as gameplay inputs.

To improve reliability and future expandability, I redesigned the interface around a Teensy 4.1. The Teensy provided more than enough GPIO, significantly better performance for encoder handling, strong USB HID support, and room for future expansion through extra GPIO, SPI, and I2C.

The PCB was designed to mount directly to the underside of the arcade control deck and connect using the JST standard already used by the machine. This made the system easier to service, easier to swap, and easier to maintain in the event of button, control, wiring, or interface-card failure.

The first revision was designed and rolled out extremely quickly, with roughly a three-week turnaround including fabrication and shipping. It was built specifically for the arcade game Tank Tonk and worked correctly on the first hardware revision. A second revision followed shortly after, generalizing the design for future BushWorks arcade machines instead of only supporting Tank Tonk.

Features

  • Teensy 4.1-based USB control interface for Raspberry Pi arcade systems.
  • USB HID output support for keyboard, gamepad, or custom HID behavior.
  • Dual quadrature optical encoder support with onboard 10 kΩ pull-up resistors to 3.3 V.
  • JST-XH connector system for organized wiring and serviceable control-deck connections.
  • Active-low button and switch input architecture.
  • Support for left and right joystick inputs.
  • Dedicated left and right button banks for player controls.
  • Top connector support for toggle switch, relay output, and 3-position momentary control states.
  • USB-powered design with no external supply required for normal operation.
  • Board designed for physical mounting under the arcade control deck.
  • Mechanical layout designed around shock, vibration, abuse, and arcade-environment serviceability.
  • Second revision expanded the design into a more general-purpose arcade interface platform.

Technical Challenges

  • Encoder reliability: the original ESP32 prototype did not handle the optical encoder inputs well enough, so the system was redesigned around the faster and more capable Teensy 4.1.
  • Fast turnaround: the arcade machine was blocked by the control interface, so the PCB had to be designed, fabricated, shipped, assembled, and deployed very quickly.
  • Arcade environment durability: the board had to survive vibration, impact, abuse, and general arcade-machine conditions.
  • Serviceability: JST-XH connectors were used so controls and wiring could be replaced or hot-swapped more easily during maintenance.
  • Fluid-spill risk: board placement was selected so spills on the control deck would be less likely to reach the electronics.
  • Mechanical mounting: the PCB was bolted to an isolation platform under the control deck to reduce shock and mounting stress.
  • Revision planning: revision one was game-specific, while revision two expanded the design into a reusable arcade control platform for future BushWorks games.
  • Firmware handoff: the board needed to be simple and well-documented enough for the software team to immediately begin writing firmware and integrating it with the Raspberry Pi.

Miscellaneous Notes

The control interface was designed as a passive electrical interface between the Teensy 4.1 and the arcade control devices. The Teensy continuously scans, debounces, and interprets the controls, then presents the resulting input state to the Raspberry Pi through USB.

The first board revision was built specifically for Tank Tonk and was successful after a single design revision. The second revision added additional I/O and generalized the design for future arcade games produced by BushWorks Entertainment.

The final board uses a 1.6 mm FR-4 PCB measuring approximately 62 × 95 mm with four M3 mounting holes and labeled connectors for maintenance.
