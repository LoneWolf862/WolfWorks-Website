Project

WolfWorks Website

Portfolio Development, Self-Hosting, Documentation, Branding, and Web Infrastructure

Active Development / Public Portfolio Platform

Short Description

WolfWorks is a self-hosted engineering portfolio and documentation platform built to collect, organize, and present years of projects under a discreet public identity. The project combines web development, server hosting, project documentation, branding, business infrastructure, and long-term creative planning.

Project Overview

WolfWorks started as a need for a professional but privacy-conscious portfolio. I wanted a place to document engineering projects, technical work, lab tools, research systems, repairs, prototypes, and personal infrastructure without placing my full personal identity at the center of the site.

Rather than using a template service, I decided to build and self-host the site myself. This required setting up a Raspberry Pi server, configuring Linux, Caddy, DNS, Cloudflare, SSL behavior, file transfer workflows, static site structure, HTML, CSS, project pages, image assets, and deployment habits.

The site also became an archival project. Many of the projects documented here were built years apart, using different tools, different machines, and different levels of documentation. Building WolfWorks required collecting old photos, finding forgotten reports, searching through GitHub repositories, reconstructing project history, and turning scattered technical memories into coherent public documentation.

The result is more than a portfolio. WolfWorks is becoming the public front end for my engineering work, business activity, future documentation, downloadable resources, video projects, and potential product development.

Features

  • Self-hosted static portfolio website.
  • Hosted on Zeta-1, a Raspberry Pi-based web server.
  • Domain, DNS, and Cloudflare configuration.
  • Caddy-based web server deployment.
  • Custom HTML and CSS site structure.
  • Dedicated project index and individual project pages.
  • Resume download and contact links.
  • Hidden expansion pages for resources and acquired wisdom.
  • Custom artwork, logo, favicon, and visual identity.
  • Project image galleries and splash images.
  • Privacy-conscious public identity built around the Wolf screen name.
  • Business-facing front end for WolfWorks Engineering.
  • Planned YouTube and documentation pipeline for future builds.

Technical Challenges

  • Self-hosting: configuring a Raspberry Pi as a reliable web server rather than relying on managed website hosting.
  • DNS and routing: connecting the domain, Cloudflare, local hosting, reverse proxy behavior, and internal network constraints.
  • Static site development: learning enough HTML and CSS to build, modify, and maintain the site directly.
  • Documentation recovery: reconstructing old projects from photos, reports, GitHub repositories, memory, and archived files.
  • Consistent formatting: creating reusable project-page structures, cards, galleries, status tags, and visual patterns.
  • Image management: standardizing naming conventions, sizing, cropping, splash images, and galleries across many projects.
  • Privacy: building a credible public portfolio while intentionally keeping personal identifying information limited.
  • AI-assisted workflow: using LLMs as formatting, editing, and development assistants without allowing them to replace the underlying project knowledge or technical judgment.
  • Long-term maintainability: designing the site so new projects, resources, downloads, and documentation can be added over time.

The Story of WolfWorks

The project began with a simple idea: I needed a portfolio. Most of my work existed in scattered places — old lab reports, GitHub repositories, phone photos, project folders, half-finished documentation, and memory. A normal résumé could not capture the scale or personality of the systems I had built.

I chose the WolfWorks name because it allowed the site to represent my work without forcing my legal identity into the foreground. The goal was to be public enough to show engineering ability, but discreet enough to maintain personal privacy and reduce unnecessary exposure.

Building the site became a project in itself. I configured the domain, set up the Raspberry Pi, installed and configured Caddy, worked through Cloudflare behavior, copied files with SCP, debugged CSS caching, created page templates, standardized image galleries, and gradually built a consistent visual identity.

AI was used throughout the process as a development assistant, formatter, editor, and brainstorming tool. The technical content, project history, decisions, and engineering judgment came from my own work; the AI helped accelerate HTML writing, layout iteration, phrasing, and organization.

WolfWorks now serves as the central archive for my projects and the foundation for future work. It connects the portfolio, business front end, contact links, resume, project pages, future resources, and planned video documentation into one coherent platform.

Miscellaneous Notes

One of the unexpected outcomes of building WolfWorks was realizing how many projects I had completed but never properly documented. Creating the site forced me to gather that history, write it down, and turn informal experience into something that can actually be shown to other people.

The site is intentionally built around real systems and real projects. Some pages are polished, some are still evolving, and some older projects have limited photos or incomplete records. That is part of the point: WolfWorks is a living archive, not just a static résumé.

In the long term, this site is intended to support project documentation, resources, downloads, potential sales, business communication, and video-based build logs through the WolfWorks YouTube channel.
